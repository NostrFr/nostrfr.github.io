Présentation générale

Oracolo propose une manière radicalement simple de publier sur le web : un fichier HTML, un npub, quelques relais, et ton blog existe. Pas de CMS, pas de base de données, pas de serveur. Ton contenu vit dans Nostr, et Oracolo ne fait qu’en révéler la forme. C’est une approche low‑tech, souveraine, qui remet la publication au niveau du protocole.

Pourquoi Oracolo ?

Parce qu’un blog ne devrait pas dépendre d’une plateforme. Parce qu’un texte ne devrait pas être enfermé dans un silo. Parce qu’une identité numérique ne devrait pas être liée à un compte, mais à une clé. Oracolo répond à ces trois besoins : publier en pair‑à‑pair, sans intermédiaire, sans friction. Un blog qui n’appartient à personne, sinon à ton npub.

Comment fonctionne Oracolo

Oracolo lit ton npub, interroge les relais que tu choisis, récupère tes événements long‑form et tes notes, puis les affiche comme un flux éditorial. Le fichier HTML contient la structure, mais le contenu vient de Nostr. Tu écris dans ton client Nostr habituel, tu signes, tu publies. Oracolo se charge du reste : il assemble, ordonne, affiche. Ton blog devient une projection de ton activité Nostr, et non un espace séparé.

Appliquer Oracolo à mon npub

J’ai relié Oracolo à mon npub pour en faire un espace personnel, vivant, minimal. Chaque publication long‑form devient un article, et chaque note peut également apparaître dans le flux. Oracolo ne distingue pas entre “blog” et “micro‑publication” : il affiche ce que je publie dans Nostr, tel quel. Chaque mise à jour est instantanée : je publie dans Nostr, mon blog se met à jour. Aucun tableau de bord, aucune interface d’édition, aucune couche supplémentaire. Juste un protocole, une clé, et un fichier HTML.

Hébergement GitHub Pages

Pour héberger Oracolo, j’ai choisi GitHub Pages : un espace statique, gratuit, transparent, versionné. Le blog n’a besoin de rien d’autre. Je pousse un fichier HTML, et l’ensemble est en ligne. Le contenu, lui, reste dans Nostr : GitHub n’héberge que la forme, jamais la donnée. C’est une séparation nette entre infrastructure et publication.

Articulation avec NostrFr

NostrFr est un hub communautaire : guides, outils, expériences, ressources pour l’écosystème francophone. Oracolo, lui, est mon espace personnel : un flux, un journal, un miroir de mon npub. Les deux se complètent : — NostrFr pour le collectif — Oracolo pour l’individuel Deux couches, un même protocole.

Pourquoi Oracolo compte

Parce qu’il montre ce que peut être un web souverain : un protocole pour publier, un fichier pour afficher, un hébergement statique pour diffuser. Pas de dépendance, pas de plateforme, pas de verrou. Juste un texte signé, relayé, visible. Un blog qui n’a besoin de rien d’autre que Nostr.
